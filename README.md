# fuzzlink

Консольная утилита нечёткого поиска строк. Принимает массив строк и поисковый запрос, возвращает N наиболее похожих строк с коэффициентом схожести. Разработана для интеграции с внешними приложениями (например, 1С) через файловый обмен JSON.

## Требования

- Python 3.12+
- [rapidfuzz](https://github.com/rapidfuzz/RapidFuzz)

## Установка зависимостей

```bash
pip install rapidfuzz pyinstaller
```

## Сборка EXE

```bash
pyinstaller --onefile --name fuzzlink app.py
```

Готовый файл: `dist/fuzzlink.exe`

## Использование

```bash
fuzzlink.exe <request_json_path> <response_json_path>
```

### Входной JSON (`request.json`)

```json
{
  "query": "оплата поставщику",
  "limit": 5,
  "items": [
    "оплата поставщика",
    "создание платежа",
    "утверждение счета",
    "платеж поставщику"
  ]
}
```

| Поле    | Тип      | Описание                              |
|---------|----------|---------------------------------------|
| `query` | `string` | Строка поиска                         |
| `limit` | `int`    | Максимальное количество результатов   |
| `items` | `array`  | Массив строк для поиска               |

- `null` и пустые строки в `items` пропускаются
- Если `limit` больше количества элементов — возвращаются все

### Выходной JSON (`response.json`)

```json
{
  "success": true,
  "results": [
    { "index": 0, "text": "оплата поставщика", "score": 94 },
    { "index": 3, "text": "платеж поставщику", "score": 88 },
    { "index": 1, "text": "создание платежа",  "score": 55 },
    { "index": 2, "text": "утверждение счета", "score": 35 }
  ]
}
```

| Поле    | Тип      | Описание                                          |
|---------|----------|---------------------------------------------------|
| `index` | `int`    | Индекс строки в исходном массиве `items`          |
| `text`  | `string` | Найденная строка                                  |
| `score` | `int`    | Коэффициент схожести от 0 до 100                  |

Результаты отсортированы по убыванию `score`.

### Формат ошибки

```json
{
  "success": false,
  "error": "текст ошибки"
}
```

Любые ошибки записываются в выходной файл — приложение никогда не падает молча.

## Алгоритм

Для каждой строки вычисляется максимум из трёх метрик [rapidfuzz](https://github.com/rapidfuzz/RapidFuzz):

| Метрика            | Что учитывает                              |
|--------------------|--------------------------------------------|
| `token_sort_ratio` | Разный порядок слов                        |
| `token_set_ratio`  | Одна строка является подмножеством другой  |
| `WRatio`           | Взвешенная комбинация нескольких алгоритмов |

Подход обеспечивает качественный поиск на русском языке.
